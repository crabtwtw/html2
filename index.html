貳 · 地雷長什麼樣

每一顆都是用真錢踩的

以下是這套系統裡，幾顆最經典的地雷。每一顆都不是「想像出來的風險」，而是真的炸過。

MINE #003

Timestamp Error — 差 500 毫秒就拒單

送出下單請求，交易所回「時間戳超出範圍」。不是你的邏輯錯，是你的伺服器時間跟交易所差了不到一秒。

沒防護的後果：用戶按「下單」，無聲無息失敗，錢沒進場，訊號過了才發現。
加了防護：自動補正 500ms 重試，用戶完全無感。
MINE #007

止損沒掛上 — API 說成功，但騙你

下單成功了，API 回傳「OK」。但止損根本沒掛上去。因為 BingX 的 stopLoss 參數要用一種特殊格式序列化，文件上寫得極模糊。

沒防護的後果：用戶以為有保險，市場反轉直接爆倉。真金白銀蒸發。
加了防護：強制 json.dumps 序列化 + 驗證止損是否真的掛上。
MINE #014

槓桿暗中縮水 — 你設 100x，他給你 50x

設定 100 倍槓桿，交易所回「設定成功」。但實際上只給你 50 倍。它不會告訴你，你要自己去查倉位才發現。

沒防護的後果：倉位金額只有預期的一半，用戶以為 bot 算錯。
加了防護：二分法探測真實最高槓桿，從設定值開始往下找到真正能用的值。
MINE #021

名義倉位上限 — 文件完全沒提

TP1 下單成功，TP2 下單失敗。原因：某些幣在高槓桿時有「隱藏上限」，超過就拒絕。這件事 BingX 文件上一個字都沒寫。

沒防護的後果：TP2 靜默失敗，用戶以為全倉都有掛 TP，結果只掛了一半。
加了防護：從錯誤訊息用 regex 抓出上限值，自動縮單到合法範圍，快取給下次用。
MINE #050

50 人同時收訊號 — 第 50 個人等 25 秒

一則訊號來了，要同時發給所有用戶。最直覺的寫法是一個一個排隊發。每個人 0.5 秒，50 人就是 25 秒。加密貨幣的世界，25 秒價格早飛了。

沒防護的後果：最後一個用戶收到訊號時進場點已過，下單直接賠。他會罵你「這破 bot 是慢半拍嗎？」
加了防護：20 條通道同時發送，所有人 1 秒內收到。像銀行從 1 個窗口變 20 個。
MINE #383

移動保本的 API 不存在 — 靜默失敗

TP1 觸發後要把止損移到保本線，保護剩餘倉位。但 BingX 的那個 API 端點已經被廢棄了，呼叫直接報錯。更可怕的是：程式靜默失敗，不通知用戶。

沒防護的後果：用戶以為保本生效了，實際上止損還在原位。市場反轉，本來該保住的利潤全吐回去。
加了防護：改用正確的 API（取消舊止損 → 下新止損單），失敗時通知用戶手動處理。

以上只是 6 顆。還有 377 顆沒講。

每一顆的故事都一樣：

寫好 → 上線 → 炸了 → 查原因 → 加保護 → 再上線

循環 383 次。

肆 · 為什麼仿不來

看得到 code，
看不到為什麼

假設有人拿到了這份 9,300 行的原始碼，他能複製嗎？

表面上能。複製貼上，部署，跑起來。

但他不知道的事：

🔒 仿不來的 5 個原因

①
他不知道為什麼有 383 個 try/except。
他會覺得「寫太多了吧，是不是不會寫才加這麼多保護？」然後他會「優化」——拿掉一堆他覺得多餘的保護。上線後，第一個用戶就遇到沒保護的 bug，系統當機。
②
他不懂交易。
「什麼是分倉？什麼是移動保本？什麼是名義倉位？」光教他搞懂這些交易概念就要 2-4 週。搞懂了也不代表他能正確實作——因為他沒有「被爆倉過」的體感。
③
他沒有 BingX 的踩坑經驗。
BingX 不是幣安，文件爛、社群小、API 會默默改版不通知。今天還能用的端點，下個月可能就廢了。這些雷沒有任何文件會寫，只能自己踩。
④
他沒有用戶回饋。
這套系統的 49 個功能，每一個都是「用戶說想要」或「用戶抱怨後加的」。工程師自己坐在電腦前想不出來「為什麼需要 V1 vs V2 勝率對決」這種功能。
⑤
他沒有實戰數據佐證。
71% 勝率、230 筆真實成交、多用戶同時運作——這不是 demo，是生產環境。任何潛在買家看到這個數據就知道：「這東西是真的能跑。」仿冒品沒有這個證明。

最致命的一點：

他拿到你的 code，
就像拿到一張掃雷地圖，
但他不知道哪些是雷。

他一改，雷就回來了。

因為那 383 個 try/except，每一個背後都有一個「為什麼要加這個」的故事。他看到 code 會想「這行多餘吧？」，然後刪掉。結果就是他刪掉了一個沙包，地雷重新露出來。

看得到 code，看不到血淚。

陸 · 護城河

真正的護城河
不是 code

code 可以被複製。但以下三樣東西不能：

第一，知道「要做什麼」。
49 個功能不是工程師想出來的。是一個懂交易、懂用戶、有商業直覺的人設計出來的。三層付費牆怎麼設？風控怎麼抓？統計怎麼分 V1 vs V2？這些是產品思維，不是技術。

第二，知道「哪裡會壞」。
383 個保護不是 paranoid。每一個都是用真錢踩過的坑。拿掉任何一個，系統就會在某個時刻、某個用戶、某個幣種上爆炸。新手不知道要加哪些，資深工程師知道但要花半年重新踩一遍。

第三，證明「能跑」。
V1 勝率 71%、230 筆真實成交、多用戶同時使用、沒有因為 bug 導致用戶爆倉。這是信用，不是功能。仿冒品做不出這個實績。
