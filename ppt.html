SECTION 02

3 步驟完成註冊

1

輸入 /start

在 Telegram 找到 Bingx 分析訂單 API 機器人，點開對話框輸入：

/start

機器人會回覆並引導您進入下一步。

2

輸入 BingX API Key

機器人會請您輸入 BingX 的 API Key 跟 Secret Key。

API KEY 怎麼取得？

登入 BingX → 點頭像 → 「API 管理」→ 建立 API → 勾選「合約交易 + 讀取」權限 → 複製 Key 跟 Secret

⚠️ 重要安全提醒

不要勾「提現」權限！只要勾「合約交易 + 讀取」就好。即使 API Key 外流，駭客也提不走您的錢。

3

等待管理員審核

送出 API Key 後，您會看到：

⏳ 你的 API 正在審核中，請耐心等待。

管理員審核通過後：

✅ 你的帳號已開通！可以開始轉傳訊號下單。

看到這則訊息表示您可以開始用了。

💡 申請被拒絕怎麼辦？

會收到：『❌ 申請未通過。如需重新申請，請輸入 /start』
意思是您可以重新跑一次申請流程，不是永久封鎖。

SECTION 04

看懂訊號卡

訊號卡有 4 個區塊，由上到下說明：

① 基本資訊

訊號發布者給的核心數據：

欄位意思
交易對哪個幣種（BTC、ETH 等）
方向📈 買漲 / 📉 買跌
止損萬一行情走錯，這個價格自動出場
止盈 1/2賺到這個價格自動部分平倉鎖利
停損幅度從進場到止損距離多少 %

② 目前設定

  • 下單環境：真實 / 模擬
  • 進場方式：立即市價（按下立刻成交）
  • 保證金模式：分倉 20 倉 = 餘額自動分 20 份用
  • 自動槓桿：系統算的安全槓桿（已加 0.5% 緩衝）

③ 帳戶資訊

分成 3 段顯示，每段都有分隔線：

第 1 段：帳戶餘額

  • 💰 真實倉可用：BingX 真實倉現在有多少 USDT 可用
  • 🌐 模擬倉可用：模擬倉餘額

第 2 段：下單試算

  • 🔓 預估保證金：這筆會用您多少錢（已扣 0.95 安全緩衝）
  • 🪙 名義倉位：保證金 × 槓桿 = 開多大的倉
  • 📦 實際數量：會買幾個幣

第 3 段：風險與獲利

  • 🧮 最多虧損：止損觸發時最多虧多少
  • 💰 TP1 達成獲利：訊號賺到 TP1 時您會賺多少
  • 💰 TP1+TP2 達成獲利：全部 TP 達成時的總獲利
💡 看訊號卡的順序

「我有多少（餘額）→ 我要投多少（保證金）→ 開多大倉（名義）→ 最壞虧多少（最多虧損）→ 順利賺多少（TP）」從上到下看，邏輯很順。

SECTION 07

設定您的偏好

3 種設定方式

A

訊號卡上直接點按鈕

每張訊號卡下方都有按鈕，可以即時切換：

  • 真實 / 模擬：切換下單環境
  • 立即市價 / 理想 D 點：切換進場方式
  • 10 倉 / 20 倉 / 50 倉 / 自訂：切換分倉數
  • 分倉 / 固定金額：切換保證金模式
  • 進階設定：展開更多選項
B

進階設定（保本模式）

保本模式意思適合誰
🔐 TP1 保本TP1 達成 → SL 移到進場價保守型
🔄 移動保本每個 TP 觸發都繼續推 SL積極型
❌ 關閉不自動移止損專業
C

每日虧損上限（風控）

設定每天虧到多少 USDT 就自動暫停所有交易。很重要的保護機制。

💡 建議數值

每日虧損上限 = 您可接受的「單日最壞情況」。例如餘額 1000，可以設 100（單日最多虧 10%）。

SECTION 09

安全保護機制

🛡️

智能槓桿（不爆倉）

市面上多數 bot 用「1 ÷ 止損% = 槓桿」，遇到大行情會強平先觸發。我們在止損% 加 0.5% 安全空間，主流幣的止損永遠先觸發。

🗂️

分倉 + 0.95 緩衝

系統把您的餘額分 20 份用，每筆訊號最多用 1 份的 95%。單筆最大虧損 ≈ 餘額的 4.75%。連虧 20 次才會傷帳戶。

🔐

移動保本

賺到 TP1 → 自動把止損推到保本價。就算後續行情反轉，也保證小賺出場。

🚨

每日虧損熔斷

單日虧損超過上限 → 系統自動暫停交易直到隔天。避免情緒化連續下單。

📊 實證數據

12 個月模擬測試，1000 個用戶，100% 帳戶都還在賺錢。對比一般 bot 同樣條件下 12 個月只剩 0% 用戶。

SECTION 10

常見問題FAQ

下單後 BingX App 顯示的數字跟訊號卡不一樣？
這是「縮量」造成的（小幣 BingX 有單筆上限）。下單結果訊息會顯示「實際使用名義倉位」跟「實際使用保證金」，後者就會等於 BingX App 看到的數字。
為什麼有時候訊號完成顯示「總獲利 0.0000 USDT」？
三種情況：1) 您手動平倉而 BingX 資料還沒同步；2) 訊號開倉超過 72 小時；3) 真的剛好打平。實際損益請以 BingX 帳戶為準。
可以手動平倉嗎？
可以。直接在 BingX App 按「一鍵平倉」。系統會在 1-3 分鐘內偵測到並發送「訊號完成」通知。不要部分平倉（平一半留一半），會讓系統誤判保本機制。
同時間多個訊號來，會一次都下嗎？
會。每個訊號各佔 1 倉。所以 20 倉模式同時下 20 筆訊號才會用完餘額。
收到「⚠️ 移動保本失敗」怎麼辦？
通常因為行情已經跑到 SL 上方（多單）或下方（空單）。不影響倉位，原本的 SL 還在守。
什麼時候用「TP1 保本」vs「移動保本」？
TP1 保本（推薦新手）：穩穩鎖獲利。移動保本：適合相信訊號會走遠的，多 TP 訊號會逐層鎖利賺更多，但有可能被中途回調掃出。
遇到「逆勢警告」訊號該下嗎？
建議跳過。逆勢訊號的勝率天生比順勢低（諧波 vs 4H 趨勢打架）。等待沒警告的訊號比較穩。
客戶被拒絕後可以重新申請嗎？
可以。看到「❌ 申請未通過。如需重新申請，請輸入 /start」直接重新打 /start 就能再次申請。
API Key 失效（換新的）怎麼辦？
聯繫管理員，他會用「🔑 刪除 API（重設）」幫您清掉舊的，您再輸入新的 API Key 即可。
收到 BingX 寄的「止損失效預警」信怎麼辦？
這是 BingX 系統自動發的標準風險提示，不是 bot 出問題。
原因：某些小幣高槓桿時，BingX 算出的強平價跟 bot 設的止損價會比較近。

處理方式：
• 倉位還在 → 手動把 SL 拉遠 0.5%，或直接平倉
• 已平倉 → 忽略此信即可

我們有「分倉保護」兜底，單筆即使爆倉也不影響帳戶整體。實測 12 個月 100% 客戶帳戶都還在。
想清自己的交易紀錄怎麼辦？
用戶端沒有自助清紀錄指令（這是設計）。原因：保留歷史 = 看到完整累計 PnL。若真的需要清除，請聯繫管理員協助。
